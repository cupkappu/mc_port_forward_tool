# Fabric 1.20.1 / 1.21.1 E2E Compatibility Design

## Goal

Adapt MC Transport Dialer to pure Fabric Minecraft `1.20.1` and `1.21.1`, then verify the target transport function with real Fabric server/client end-to-end tests.

## Scope

In scope:

- Build this mod for Minecraft `1.20.1` and `1.21.1`.
- Use Fabric Loader `0.19.3` for both target versions.
- Use Fabric API `0.92.9+1.20.1` for `1.20.1`.
- Use Fabric API `0.116.12+1.21.1` for `1.21.1`.
- Keep one shared transport core for protocol, crypto, config, TCP listener, stream registry, backpressure, and target TCP forwarding.
- Split only the Minecraft/Fabric API integration into version-specific adapters.
- Run real server/client tests against minimal pure Fabric environments.

Out of scope:

- Forge and NeoForge.
- Auto-testing full third-party modpacks.
- SOCKS5, HTTP CONNECT, UDP, TUN, DNS, routing, GUI, or arbitrary server target dialing.
- Supporting older Fabric versions such as `1.19.2`.

## Architecture

### Shared Core

The shared core remains under `src/main/java/dev/kifuko/mctransport` and must not import `net.minecraft.*` or `net.fabricmc.*`, except for the server/client initializer classes that are moved into version adapters.

Shared core owns:

- `Frame`, `FrameCodec`, `SecureFrameCodec`, `FrameType`
- `PskCipher`
- config models and loader
- TCP listener and stream implementations
- stream registry and buffer budget
- `TunnelBridge`

### Version-Specific Fabric Adapters

Create version adapter source roots:

- `src/fabric1201/java`
- `src/fabric1211/java`

Each target source root provides these same FQCNs:

- `dev.kifuko.mctransport.client.McTransportClient`
- `dev.kifuko.mctransport.client.FabricClientTunnelBridge`
- `dev.kifuko.mctransport.client.TransportPayload` if needed by that version
- `dev.kifuko.mctransport.server.McTransportServer`
- `dev.kifuko.mctransport.server.FabricServerTunnelBridge`

Only one adapter source root is included per build, selected by `-PtargetMinecraft=1.20.1` or `-PtargetMinecraft=1.21.1`.

### Build Selection

Add version property files:

- `versions/1.20.1.properties`
- `versions/1.21.1.properties`

Gradle loads the selected file and configures:

- `minecraft_version`
- `loader_version`
- `fabric_api_version`
- `loom_version`
- `java_language_level`
- `adapter_source_set`
- `minecraft_dependency_range`
- `fabric_api_dependency_range`

Expected commands:

```bash
./gradlew clean build -PtargetMinecraft=1.20.1
./gradlew clean build -PtargetMinecraft=1.21.1
```

Expected artifacts:

```text
build/libs/mc-transport-dialer-1.20.1-0.1.0.jar
build/libs/mc-transport-dialer-1.21.1-0.1.0.jar
```

## E2E Test Design

Create `scripts/e2e/` helpers for real pure-Fabric environments:

- `scripts/e2e/prepare_fabric_server.sh <version> <workdir>`
- `scripts/e2e/write_configs.sh <server-dir> <client-dir> <uuid> <psk>`
- `scripts/e2e/echo_server.py`
- `scripts/e2e/tcp_probe.py`

The automated part can:

1. Prepare a clean Fabric server directory.
2. Install Fabric server launcher, Fabric API, and this mod jar.
3. Write server config with `target_host=127.0.0.1`, `target_port=10000`, whitelisted UUID, and PSK.
4. Start server in nogui mode.
5. Start the server-side echo listener.

The client remains a real GUI Minecraft client step because joining a real server requires an authenticated game session. The manual step is:

1. Install the matching jar and Fabric API into a real client instance.
2. Write client config with the same PSK.
3. Join the prepared server.
4. Run `scripts/e2e/tcp_probe.py 127.0.0.1 25580`.

Pass criteria:

- Client log says local listener bound to `127.0.0.1:25580`.
- Server log says the player session is ready.
- TCP probe receives the exact bytes it sent.
- Two concurrent probes both receive exact echo bytes.
- Removing the UUID from server config prevents transport auth.

## Error Handling

- If a target version fails to compile due to Fabric networking API drift, fix only that version adapter.
- If shared core tests fail, fix shared core once and rerun both versions.
- If server starts but client transport fails, check PSK, UUID, clock skew, and channel registration logs before changing code.
- If `tcp_probe.py` connects but times out, inspect server target dial and stream OPEN/DATA logs.

## Verification Gates

Required before completion:

- `./gradlew test -PtargetMinecraft=1.20.1`
- `./gradlew test -PtargetMinecraft=1.21.1`
- `./gradlew build -PtargetMinecraft=1.20.1`
- `./gradlew build -PtargetMinecraft=1.21.1`
- Real Fabric server/client TCP probe pass on `1.20.1`.
- Real Fabric server/client TCP probe pass on `1.21.1`.

Completion is not proven by unit tests alone; at least one real client must join one real server for each target version and pass the TCP echo probe.
